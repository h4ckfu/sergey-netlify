When making templates try to enforce a core structure, but avoid enforcing a particular context. Most uses of the template have a different context. Do this for every template used by one of these templates, and so on.

Use block variables to specify sections of your template that people are likely to change; Usually where they want to replace their own content.Core templates should ideally be simple generic components that can be used in other templates to create more complex page layouts.

They should be so flexible that developers and themes can easily use them without having to replace the template. The templates should try to encapsulate a core structure for the element as well as key classes, while the content can be easily overridden.

Finally, we want to avoid duplicating HTML from template to template, especially if there are some classes associated with the HTML element. If there are any, repeat the above steps for them as well.

Write a test page that uses your template to identify some of the assumptions that might appear in the template. The sample context that you provide in the template is mainly used to display the template in the template library, and it's probably not what your template actually is.
